## 功能分支工作流

### 1. 开始开发新功能

```bash
# 从main分支创建新的功能分支
git checkout main
git pull origin main
git checkout -b feature/新功能名称
```

### 2. 在功能分支上开发

```bash
# 切换到功能分支
git checkout feature/ai-chat

# 修改代码...
git add .
git commit -m "feat(ai-chat): 添加多模型支持"
```

### 3. 推送到远程仓库

```bash
# 推送分支到GitHub
git push origin feature/ai-chat
```

### 4. 创建Pull Request

1. 在GitHub上创建Pull Request
2. 邀请同事进行代码审查
3. 确保所有测试通过
4. 合并到main分支

### 5. 合并后清理

```bash
# 切换回main并拉取最新代码
git checkout main
git pull origin main

# 删除已合并的分支
git branch -d feature/ai-chat
git push origin --delete feature/ai-chat
```

## 分支命名规范

| 类型 | 前缀 | 示例 |
|------|------|------|
| 新功能 | feature/ | feature/ai-chat |
| Bug修复 | fix/ | fix/chat-error |
| UI改进 | ui/ | ui/homepage-redesign |
| 重构 | refactor/ | refactor/api-service |
| 文档 | docs/ | docs/api-guide |

## 提交信息规范

```
<type>(<scope>): <subject>

[optional body]

[optional footer]
```

### 类型(type)
- feat: 新功能
- fix: Bug修复
- docs: 文档更新
- style: 代码格式调整
- refactor: 重构
- test: 测试相关
- chore: 构建/工具

### 示例
```
feat(ai-chat): 添加多模型切换功能

- 支持GPT-4、Claude、Gemini等模型
- 添加模型选择下拉菜单
- 优化响应速度

Closes #123
```
